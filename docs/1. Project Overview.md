# Project Overview

## What is Planner Redesign?

Planner Redesign is a **web-based planning and scheduling optimization tool** that uses constraint programming to generate optimal test schedules. It combines a browser-based user interface with a Python-based optimization solver to help teams plan, configure, and execute complex scheduling scenarios.

## Core Purpose

The system solves the **resource-constrained project scheduling problem (RCPSP)** for hardware/software testing workflows. It enables:

- **Intelligent Scheduling**: Automatically generate optimal schedules respecting temporal constraints, resource availability, and priorities
- **Multi-objective Optimization**: Support for various priority modes (leg priority, end-date priority, resource bottleneck optimization, etc.)
- **Batch Execution**: Run multiple scenarios with different configurations
- **Real-time Visualization**: Gantt charts, resource utilization graphs, and concurrency timelines
- **Data-driven Configuration**: Upload test data, configure constraints, and analyze results

- Upload and edit CSV data files (legs, tests, FTE, equipment)
- Validate data integrity before solving
- Preview and transform data

### Configuration
- Multi-mode priority configuration (6 priority modes)
- Resource constraint settings
- Deadline and proximity rule definitions
- Scenario-specific overrides

### Solver
- CP-SAT constraint programming engine
- Real-time progress streaming via SSE (Server-Sent Events)
- Configurable time limits and debug levels
- Batch scenario execution

### Visualization
- Interactive Gantt charts for test schedules
- Resource utilization timelines
- FTE and equipment usage graphs
- Concurrency analysis

### Output
- Schedule CSV exports
- Resource utilization reports
- Performance metrics and diagnostics
- Execution history and run management

## Getting Started

### Quick Start

```bash
# Install all dependencies
./install.sh

# Start all services
./start.sh
```

**Default Ports:**
- Frontend: http://localhost:3000
- Backend API: http://localhost:8000
- API Documentation: http://localhost:8000/docs

### Manual Setup

```bash
# Frontend
cd frontend && npm install && npm run serve

# Backend
cd backend && python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
uvicorn src.api.main:app --reload --port 8000

# Solver (standalone CLI)
cd solver && pip install -r requirements.txt
python main.py --input-folder ../input_data/my_scenario
```

### Running Tests

```bash
# Frontend tests
cd frontend && npm test

# Backend tests
cd backend && python -m pytest

# Solver tests
cd solver && python -m pytest
```

## Architecture Summary

The system follows a **three-tier architecture**:

1. **Frontend (Presentation Layer)**: Browser-based SPA using Alpine.js for reactive components. Communicates with backend via REST API and Server-Sent Events for real-time updates.

2. **Backend (Service Layer)**: FastAPI application exposing REST endpoints. Handles session management, file operations, and orchestrates solver execution. Manages in-memory state for active runs.

3. **Solver (Optimization Layer)**: Standalone Python module using Google OR-Tools CP-SAT solver. Can run as CLI tool or be invoked by backend service. Implements constraint programming model with multiple priority modes.

**Key Architectural Patterns:**
- **Event-Driven UI**: Alpine.js reactive stores with HTMX for partial updates
- **Service Layer Pattern**: Backend uses service classes with dependency injection
- **Orchestrator Pattern**: Solver pipeline orchestrated by central coordinator
- **Repository Pattern**: File operations abstracted behind service interfaces
