## Your Task: Implement Backend API Contracts

### 1. Spreadsheet Discovery Endpoint
Add GET /api/spreadsheets/discover
- Scan configured upload paths and session uploads
- Return list of CSV/XLSX/XLS files with stable identifiers
- Include metadata (size, last modified, row count estimate)

### 2. Spreadsheet Validation Endpoint
Add POST /api/spreadsheets/validate
- Validate required columns: project, leg, branch, test, duration_days, description, next_leg
- Return actionable errors with row/column identification
- Return extracted entities: projects, leg types, leg names, test types, computed test names

### 3. Config Consistency Check Endpoint
Add POST /api/config/consistency
- Validate imported JSON config against active spreadsheet entities
- Return warnings for out-of-scope references

### 4. Scenario Queue Orchestration API
Add endpoints under /api/scenarios/queue:
- POST /add - add scenario to queue
- POST /run-one - run single scenario
- POST /run-all-unsolved - run all unsolved
- GET /status - get queue status
- POST /stop-render - stop rendering, preserve state

## Requirements
- Follow existing backend patterns in the codebase
- Add proper error handling
- Define TypeScript types for all request/response shapes
- Do NOT implement full logic - focus on API contracts and wire-up
- Verify endpoints are reachable after implementation
